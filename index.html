HTML, czyli opakowanie dla treści.

Na początek podstawa podstaw, czyli Hyper Text Markup Language. Nie jest to język programowania, a język znaczników, które służą za odpowiednie opakowanie treści. Niczym kulturysta opakowujący swoje ciało mięśniami.

Znaczniki/TAGI/Elementy

Nie ma HTML-a bez znaczników. Są to formuły pomiędzy którymi zawarta jest treść i nadają one jej odpowiednie znaczenie. Np znacznik “h1” oznacza, że tekst zawarty pomiędzy tymi znacznikami jest nagłówkiem poziomu pierwszego. Na tej stronie tym znacznikiem oznaczony jest tekst: “Robert Uba - Strona Domowa”. Znaczniki wpisujemy w ostrych nawiasach (<> ), które są bardzo znanym elementem graficznym związanym z programowaniem.

CSS-y

W następnej lekcji poznaję tzw. style czyli CSS. Cascade Style Sheet - Kaskadowe arkusze stylów. CSS odpowiada za prezentację ( wygląd ) treści na stronie. Zbudowany jest z reguł. Składnia wygląda o tak:

Każdy w IT jest GIT

Git czyli system kontroli wersji napisany, w dosłownie kilka dni(!) przez twórcę Linuxa - Linusa Torvaldsa. Obecnie jest standardem i każdy twórca oprogramowania winien umieć z niego korzystać. Jest to narzędzie służące, mówiąc w skrócie, do zabezpieczania starszych wersji naszego projektu. Dzięki niemu możemy zapobiegać różnorakim problemom, które mogą wystąpić w trakcie tworzenia oprogramowania takich jak: program przestał działać, a wczoraj było git; skasowałem przypadkiem plik; gdybym miał poprzednią wersję…etc. Jest bardzo pomocny w pracy, bo mając świadomość zapisania ostatniej wersji ułatwia dalszą pracę nad kodem, bez strachu, że nie uda wrócić się do stanu poprzedniego gdyby coś poszło nie tak. Kolejną strefą zastosowania jest praca grupowa, gdzie dzięki gitowi łatwiej utrzymać porządek w projekcie. Git instaluje się na komputerze. Obecnie oprócz zwykłego GIT-aktórej używa się poprzez terminal systemowy, jest także wersja Git desktop, która staje się obecnie standardem i jest wystarczająca do większości zastosowań. Korzystanie z gita polega w skrócie na Commitowaniu zmian jakie zaszły w projekcie, czyli np. wprowadzenie nowego artykułu na stronie, czy znaczące zmiany w kodzie.
